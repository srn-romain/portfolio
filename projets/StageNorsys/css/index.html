Les Locaux

Git est un système de contrôle de version largement utilisé dans les entreprises, y compris chez Norsys. Voici maintenant deux fonctionnalités couramment utilisées avec Git : la création d'une MR et la création d'une branche.

Effectuer une MR avec Git

Comme nous l'illustre la capture d'écran ci-dessus, voici comment effectuer une merge request (MR) sur l'espace distant de Norsys en utilisant Git :


  1. Utilisez la commande "git checkout"" pour vous placer sur votre branche locale où vous avez effectué vos modifications.
  2. Effectuez un "git add" pour ajouter les fichiers modifiés à l'index, puis utilisez "git commit" pour créer un commit contenant vos modifications.
  3. Utilisez "git push" pour pousser votre branche locale vers le référentiel distant.
  4. Accédez à la plateforme de gestion du code source : GitLab
  5. Dans la MR, sélectionnez la branche cible (branche de destination) vers laquelle vous souhaitez fusionner vos modifications.
  6. Ajoutez un titre descriptif à la MR et fournissez une description claire et brève en expliquant les modifications effectuées.
  7. Attendez que les relecteurs passent en revue votre MR. La pipeline est en cours...
  8. Une fois que votre MR a été approuvée, votre branche est mergée !



Créé une nouvelle branche avec Git

Nous allons voir dans cette troisième partie comment créer une nouvelle branche de code en utilisant Git :


  1. Ouvrez un terminal depuis un IDE, InteliJ dans notre cas.
  2. Votre branche sera créée à partir de votre emplacement actuel, qui est généralement l'inté.
  3. Réaliser ensuite la commande suivante : git checkout -b "nom_de_la_branche"
  4. Votre branche est maintenant créée ! Par défaut vous serez automatiquement placer dedans.
  5. Il est ensuite conseillé dans notre environnement de faire un "npm run generate:api" pour regénéré automatiquement le code nécessaire.
  6. Pour basculer vers une autre branche, vous pouvez utiliser la commande git checkout "nom_branche"

Outil NPM

Projet CRM

NPM, acronyme de "Node Package Manager", est un gestionnaire de packages (ou bibliothèques de code) pour l'écosystème Node.js. Il est utilisé pour installer, gérer et partager des packages JavaScript réutilisables.

Installation de Node.js : NPM est généralement installé automatiquement lors de l'installation de Node.js. Lorsque vous installez Node.js, NPM est livré avec en tant que gestionnaire de packages par défaut.

Initialisation d'un projet : Vous pouvez utiliser NPM pour initialiser un nouveau projet Node.js en créant un fichier package.json. Ce fichier contient des informations sur votre projet, telles que son nom, sa version, ses dépendances, etc. Vous pouvez l'initialiser en utilisant la commande npm init dans le répertoire de votre projet.

Utilisation de NPM dans le projet : Une fois que vous avez configuré le projet Node.js, vous pouvez utiliser les fonctionnalités de NPM directement depuis l'IDE (InteliJ dans notre cas). Pour cela, rendez-vous dans la fenêtre "Terminal" et exécuter des commandes NPM comme vous le feriez en ligne de commande. En voici quelques exemples :

  • npm install: Cette commande installe tous les packages répertoriés dans le fichier package.json du projet.
  • npm install 'package': Cela installe un package spécifique dans le projet.
  • npm update: Cette commande met à jour tous les packages du projet vers leurs dernières versions disponibles.
  • npm uninstall 'package': Cela supprime un package spécifique du projet.
  • npm run 'script': Cela exécute un script spécifique défini dans le fichier package.json du projet. C'est le cas sur la capture d'écran ci-dessus.


En résumé, NPM est un gestionnaire de packages pour NodeJS qui permet d'installer, gérer et partager des packages JavaScript. Il facilite le développement en fournissant un moyen simple de gérer les dépendances de votre projet et en offrant un accès facile à un vaste écosystème de packages JavaScript.     Grâce à son interface intuitive et conviviale, GitKraken simplifie le processus de développement et améliore la productivité des développeurs chez Norsys. L'outil offre également des fonctionnalités avancées telles que la résolution visuelle des conflits et la possibilité de remonter dans l'historique des modifications, ce qui facilite le travail en équipe et la révision du code.
    En intégrant GitKraken dans leur flux de travail, Norsys peut assurer une collaboration harmonieuse, une meilleure traçabilité des modifications et une gestion efficace des versions de leurs projets.


    Enfin, GitKraken offre une intégration transparente avec des plateformes de gestion de code populaires telles que GitHub, Bitbucket et GitLab, ce qui permet à Norsys de travailler de manière transparente avec des dépôts distants et d'exploiter pleinement les avantages du développement basé sur Git. Une fois mon ticket réalisé en local (encadrer jaune), il est nécessaire de fusionner le projet sur la branche principale, appelée "inte" (encadrer rouge), afin de rendre les modifications accessibles aux autres développeurs. Cette étape permet d'assurer la cohésion et la collaboration au sein de l'équipe de développement.


    A quoi servent les boutons encadrer en bleu ?
    -Le bouton "Pull" permet de récupérer les dernières modifications du référentiel distant vers votre copie locale. Cela synchronise votre travail avec les mises à jour effectuées par d'autres contributeurs.
    -Le bouton "Push" vous permet d'envoyer vos modifications locales vers le référentiel distant. Cela met à jour le référentiel partagé avec vos contributions.
    -Le bouton "Branch" permet de créer une nouvelle branche. Un developpeur l'utilise lorsqu'il doit réaliser une task par exemple
    -"Stash" est un mécanisme pratique pour mettre de côté temporairement des modifications non validées. Cela vous permet de basculer rapidement vers une autre branche sans commettre vos changements en cours.
    -Une fois que vous souhaitez réappliquer vos modifications, vous pouvez utiliser le bouton "Pop" pour les récupérer facilement.
    -Le bouton "Terminal" ouvre une interface en ligne de commande intégrée où vous pouvez exécuter des commandes Git directement depuis Git Kraken. Cela permet d'accéder à des fonctionnalités plus avancées et de gérer votre référentiel à l'aide de commandes spécifiques à Git.
